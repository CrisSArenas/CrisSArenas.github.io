TinyML Model Quantization Lab

Understanding This Lab

What This Tool Does

This lab demonstrates neural network quantization for TinyML deployment. It shows how reducing weight precision from 32-bit floating point (FP32) to 8-bit integer (INT8) or 4-bit integer (INT4) dramatically reduces model size and inference time, enabling deployment on resource-constrained microcontrollers.

Weight Distribution Heatmap

Each cell in the heatmap represents a neural network weight. Blue indicates negative values, white is near-zero, and red indicates positive values. With FP32, you see smooth gradients with many unique values. INT8 quantization snaps weights to 256 discrete levels. INT4 uses only 16 levels, producing a visibly blocky pattern.

Quantization Metrics

Scale Factor
Multiplier that maps floating-point range to integer range. Computed as (max - min) / (2^bits - 1).
Zero-Point
Integer value that corresponds to floating-point zero. Ensures zero is exactly representable.
Accuracy
Classification accuracy on a test set. Drops slightly with quantization due to precision loss.
Model Size
Total model size in KB. Reduces proportionally with bit-width reduction.
Inference Time
Time to run one forward pass on a Cortex-M7 MCU. INT8/INT4 ops are much faster than FP32.
Flash Usage
Percentage of MCU flash memory occupied by the model plus application code and bootloader.

MCU Flash Deployment

The flash memory visualization shows how a 512KB STM32H7 flash is partitioned. The bootloader (16KB) and application code (48KB) are fixed. The model segment varies with quantization level. If the model exceeds available flash, the bar turns red indicating the model will not fit.

Model Architectures

MobileNet-v2 (Tiny)
Lightweight architecture optimized for mobile and edge devices. Uses depthwise separable convolutions. Good accuracy-to-size ratio.
Custom CNN (3-layer)
Simple 3-layer convolutional network. Smallest model, highest base accuracy on simple tasks like MNIST digit classification.
ResNet-8
Compact residual network with skip connections. Balances depth and size, robust to quantization thanks to residual paths.

How This Lab Works

STEP 1

Pick a Model & Precision

Select a model architecture (MobileNet, ResNet, or Custom CNN) and a quantization level (FP32, INT8, or INT4). Watch the weight heatmap change in real time.

STEP 2

Flash to MCU

Click Flash to MCU to deploy the model into simulated STM32 flash memory. The progress bar shows each stage of the flash process.

STEP 3

Draw or Load a Digit

After flashing, draw a digit (0-9) on the black canvas with your mouse or finger, or click Random Sample to load one automatically.

STEP 4

Run Inference

Click Run and watch the forward pass animate through each layer. The network classifies your digit and shows confidence scores per class.
