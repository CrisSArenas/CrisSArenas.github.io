System Overview

The GenAI Portfolio Commentary Engine is an LLM-driven narrative generation system designed to automate institutional-grade investment commentary. It ingests structured portfolio, performance, and market data, processes it through a multi-phase NLP pipeline, and generates high-quality commentary aligned with internal compliance, tone, and domain expertise.

The engine integrates:

  • A feature extraction layer
  • A rules-driven financial insight generator
  • A multi-stage LLM reasoning chain
  • Guardrails ensuring factual, numerical, and regulatory correctness
  • Automated PDF/HTML reporting modules

It replaced an entirely manual process typically requiring hours of analyst time per client.

Data Engineering & Feature Pipeline

Data Sources

The system processes:

  • Daily/weekly portfolio performance data
  • Attribution summaries (sector, factor, style drivers)
  • Benchmark returns
  • Asset-class breakouts
  • Exposure changes
  • Manager/fund-level deltas

All inputs are version-controlled and stored in Parquet for reproducibility.

Feature Engineering

A dedicated transformer maps raw data into domain-specific features used by the LLM:

  • Return decomposition (active/passive drivers)
  • Top contributors & detractors
  • Rolling volatility trends
  • Factor beta changes
  • Exposure rotations
  • Liquidity signals
  • Cross-portfolio consistency checks

Each feature is encoded into a structured JSON artifact:

{ "period": "Q3 2025", "portfolio_return": 5.45, "benchmark_return": 4.90, "active_contribution": 0.55, "exposure_changes": {...}, "top_drivers": [...], "risk_flags": [...], "style_shifts": {...} }

Insight Generation Engine

Before hitting the LLM, the system runs a deterministic layer to ensure all insights are numerically accurate and material deltas exceeding thresholds are flagged.

Rule Types

  • Return significance rule: identify statistically meaningful deviations
  • Sector drift rule: detect >2% absolute moves
  • Alpha decomposition rule: segment active return components
  • Liquidity rule: highlight sudden liquidity changes in DMA portfolios

This ensures that the LLM receives not raw numbers, but interpreted financial signals.

LLM Reasoning Chain Architecture

Multi-Stage Pipeline

The commentary engine uses a 4-stage LLM pipeline:

  1. Draft generation (structured → natural language)
  2. Tone normalization (institutional investment tone)
  3. Compliance & factuality enforcement
  4. Formatting & summarization

Each stage is treated as a separate "agent" with specific responsibilities.

Prompt Engineering Design

Prompts use a controlled structure:

  • JSON inputs
  • Strict constraints ("do not fabricate values," "use only provided data")
  • Required sections (performance, attribution, exposures, risk flags)
  • Fintech domain vocabulary
  • Style guide (formal, concise, actionable)
Example constraint: "You must ONLY use the numerical values provided. If information is missing, state that the data is unavailable. NEVER invent data or refer to market moves outside the scope."

Guardrails

The system implements:

  • Numeric reconciliation checks
  • Semantic similarity validation
  • Disallowed phrase filters
  • Fact-alignment scoring

Outputs failing validation are auto-routed back to regeneration with modified constraints.

RAG (Retrieval-Augmented Generation) Layer

Optional but supported:

  • Embedding model encodes historical commentary
  • Vector search retrieves stylistically similar past narratives
  • Retrieved snippets guide consistency
