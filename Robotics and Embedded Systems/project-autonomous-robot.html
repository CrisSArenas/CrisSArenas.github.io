Autonomous Mobile Robot

2025 - 2026
Embedded Robotics

A differential-drive autonomous mobile robot built from scratch, running ROS 2 on a Raspberry Pi 4. Integrates RPLiDAR A1 for 2D SLAM via Cartographer, BNO055 IMU for orientation estimation, and closed-loop motor control with encoder feedback. Navigates autonomously through Nav2 with real-time obstacle avoidance.

ROS 2 Nav2 Cartographer C++ Python Raspberry Pi 4 RPLiDAR A1 BNO055

System Overview

This project is a ground-up build of a differential-drive autonomous mobile robot. The compute platform is a Raspberry Pi 4 running Ubuntu and ROS 2 (Humble). The robot integrates multiple sensor modalities for localization, mapping, and navigation:

  • RPLiDAR A1 for 360-degree 2D laser scanning
  • BNO055 9-axis IMU over I2C for orientation estimation
  • Wheel encoders for odometry and closed-loop motor control
  • DC motors with H-bridge driver for differential drive

All sensor reads and motor control are handled by custom C++ ROS 2 nodes. The system performs SLAM using Google Cartographer and autonomous navigation through the Nav2 stack.

Hardware Integration

RPLiDAR A1 — 2D Laser Scanner

The RPLiDAR A1 provides 360-degree scanning at up to 8000 samples per second with a range of 12 meters. Connected over USB to the Raspberry Pi, the custom C++ node publishes /scan messages (sensor_msgs/LaserScan) for consumption by Cartographer and the Nav2 costmap layers.

BNO055 IMU — Orientation Estimation

The Bosch BNO055 is a 9-axis absolute orientation sensor (accelerometer, gyroscope, magnetometer) with on-chip sensor fusion. Connected over I2C to the Raspberry Pi GPIO header, the custom C++ node reads quaternion orientation data at 100 Hz and publishes to /imu/data (sensor_msgs/Imu). The IMU data feeds into both Cartographer for improved scan matching and Nav2 for robot orientation.

Motor Control & Encoders

Dual DC motors driven through an L298N H-bridge with PWM speed control via GPIO. Optical wheel encoders provide tick counts for odometry computation. The motor control node subscribes to /cmd_vel (geometry_msgs/Twist), converts linear and angular velocities to individual wheel speeds, and runs a PID controller per wheel to maintain target RPM.

SLAM & Navigation

Cartographer SLAM

Google Cartographer performs real-time 2D SLAM by fusing LiDAR scans with IMU data. The system builds an occupancy grid map while simultaneously localizing the robot within it. Key configuration parameters:

  • Submaps: 90 range data entries per submap for dense local maps
  • Scan matching: Ceres-based optimization with IMU gravity constraint
  • Loop closure: Branch-and-bound scan matching for global consistency
  • Map resolution: 5cm grid cells for navigation-quality maps

Nav2 Autonomous Navigation

The Nav2 stack handles goal-directed autonomous navigation with:

  • Global planner: NavFn (Dijkstra/A*) for optimal path computation on the static costmap
  • Local controller: DWB (Dynamic Window Approach) for real-time velocity commands respecting dynamic obstacles
  • Costmap layers: Static layer (from SLAM map), obstacle layer (live LiDAR), inflation layer (safety margins)
  • Recovery behaviors: Spin, back-up, and wait behaviors for stuck detection and recovery
  • Behavior tree: NavigateToPose BT orchestrates the full navigation pipeline

Control System

PID Motor Control

Each wheel runs an independent PID controller that compares the target RPM (derived from /cmd_vel) against actual RPM (measured from encoder ticks). The controller outputs a PWM duty cycle adjustment at 50 Hz.

  • Proportional gain (Kp): Corrects instantaneous speed error
  • Integral gain (Ki): Eliminates steady-state error from friction and load
  • Derivative gain (Kd): Dampens oscillation during acceleration/deceleration
Result: PID-tuned to less than 5% steady-state error on both wheels, with smooth acceleration profiles and minimal overshoot during direction changes.

Odometry

Wheel odometry is computed from encoder tick differentials using the differential-drive kinematic model. Position (x, y) and heading (theta) are integrated at 50 Hz and published as /odom (nav_msgs/Odometry) with the corresponding TF transform (odom → base_link).

Results

The completed robot demonstrates:

  • Real-time SLAM with centimeter-accuracy maps in indoor environments
  • Autonomous navigation to arbitrary goal poses with dynamic obstacle avoidance
  • Less than 5% steady-state error on closed-loop motor control
  • Reliable I2C communication with the BNO055 IMU at 100 Hz
  • Full ROS 2 integration with standard message types and TF tree
