FreeRTOS Concepts Guide

What This Simulator Does

This interactive simulator demonstrates a FreeRTOS-style preemptive priority scheduler. It models real-time tasks with configurable periods and execution times, mutex-based resource sharing with optional priority inheritance, and counting semaphores for producer-consumer patterns. The Gantt chart visualizes scheduling decisions tick-by-tick.

Task States

READY
Task is eligible to run and waiting in the ready queue. The scheduler picks the highest-priority ready task.
RUNNING
Task is currently executing on the CPU. Only one task runs at a time.
BLOCKED
Task is waiting for an event: a delay expiry (vTaskDelay), a mutex (xSemaphoreTake), or a semaphore.
SUSPENDED
Task is paused via vTaskSuspend and will not run until vTaskResume is called.

Priority Inversion

Occurs when a high-priority task is blocked waiting for a mutex held by a low-priority task, while a medium-priority task preempts the low-priority task. This causes the high-priority task to wait indefinitely (unbounded inversion).

Priority Inheritance Protocol solves this by temporarily boosting the low-priority task's priority to match the blocked high-priority task, preventing the medium-priority task from preempting it.

Scheduling Algorithm

Preemptive
A higher-priority task that becomes ready will immediately preempt the currently running task.
Round-Robin
Tasks at the same priority level share CPU time in round-robin fashion.
Periodic Tasks
Tasks with a non-zero period are reactivated every T ticks. If a task does not complete before its next activation, a deadline miss is recorded.

How to Use This Simulator

STEP 1

Select a Preset

Choose a scenario: Basic Scheduling, Priority Inversion, Inheritance, Deadline Miss, or Producer-Consumer.

STEP 2

Play the Simulation

Press Play or Space to start. Use Step for tick-by-tick execution to study each scheduling decision.

STEP 3

Watch the Gantt Chart

Observe the Gantt chart fill as tasks execute, get preempted, and block on resources. Colors show each task's state.

STEP 4

Toggle Priority Inheritance

Enable Priority Inheritance to see how it resolves priority inversion. Compare the Gantt charts before and after.

STEP 5

Experiment

Add custom tasks, mutexes, and semaphores to build your own scenarios and learn scheduling behavior.
